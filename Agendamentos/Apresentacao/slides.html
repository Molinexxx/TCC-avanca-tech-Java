Slide 1

Sistema de Agendamentos em Java com Spring Boot

Projeto atualizado para uma versao mais profissional, com backend organizado, interface web moderna e execucao simples pelo IntelliJ.

Pronto para demonstracao

Cadastro, validacao, consulta da agenda do dia e banco H2 integrados em uma unica aplicacao.

Slide 2

Problema e objetivo

  • O projeto inicial tinha a base Java montada, mas ainda sem acabamento para apresentacao.
  • Faltava uma interface web mais bonita e uma estrutura mais clara no backend.
  • O objetivo foi transformar a base em um sistema pronto para abrir, rodar e demonstrar.

Resultado esperado

Uma aplicacao funcional, organizada e visualmente forte para uso academico ou demonstracao tecnica.

Slide 3

Arquitetura da aplicacao

  • Controller: recebe as requisicoes HTTP.
  • Service: aplica as regras de negocio.
  • Repository: faz o acesso ao banco H2.
  • Entity: representa os dados do agendamento.
  • Static: entrega a interface HTML, CSS e JavaScript.

Separacao de responsabilidades

Cada camada cuida de uma parte do sistema, o que melhora manutencao, leitura e evolucao futura.

Slide 4

Fluxo de funcionamento

  1. O usuario abre o sistema em http://localhost:8080.
  2. Preenche os dados do agendamento na tela inicial.
  3. O frontend envia os dados para a API do Spring Boot.
  4. O backend valida e salva o registro no H2.
  5. A agenda do dia e atualizada automaticamente.

Beneficio

O fluxo ficou simples para o usuario e facil de explicar durante a apresentacao.

Slide 5

Melhorias implementadas

  • Padronizacao da estrutura Java.
  • Correcao de nomes e campos do modelo.
  • Validacoes obrigatorias no backend.
  • Bloqueio de conflito de horario por profissional.
  • Tratamento de erros com mensagens amigaveis.
  • Tela inicial moderna para cadastro e consulta.

Destaque

O sistema agora nao e so funcional. Ele tambem transmite mais qualidade visual e tecnica.

Slide 6

Tecnologias utilizadas

Java 17
Spring Boot 3.2.5
Spring Web
Spring Data JPA
Bean Validation
H2 Database
HTML, CSS e JavaScript
Maven

Motivo da escolha

Essas tecnologias permitem desenvolver uma aplicacao completa, leve e facil de executar localmente.

Slide 7

Como executar no IntelliJ

1. Abrir a pasta Agendamentos no IntelliJ 2. Esperar o Maven importar as dependencias 3. Rodar a classe AgendadorHorariosApplication 4. Abrir http://localhost:8080 5. Opcional: acessar http://localhost:8080/h2-console

Tambem foi criada uma configuracao de execucao para facilitar esse processo.

Slide 8

Demonstracao sugerida

  1. Mostrar a tela inicial.
  2. Cadastrar um novo agendamento.
  3. Exibir a agenda do dia atualizada.
  4. Tentar repetir o mesmo horario para o mesmo profissional.
  5. Apresentar a validacao impedindo conflito.

Mensagem forte para apresentar

O sistema organiza atendimentos, evita erros operacionais e oferece uma experiencia visual melhor para o usuario.

Slide 9

Conclusao

O projeto foi recriado e atualizado para uma versao mais completa, com codigo mais limpo, melhor organizacao e uma interface pronta para demonstracao.

Isso mostra nao apenas o funcionamento tecnico do sistema, mas tambem uma preocupacao com usabilidade, estrutura e apresentacao profissional.

Entrega final

Backend funcional, frontend integrado, testes passando e material pronto para apresentar.
